Projects

Geopresence

  • The main idea behind this work is to create a geospatial index and query system for edge computing.

    For a simplistic example, image air quality and weather monitoring devices placed on streetlights in a city; instead of querying weather.com query the streetlight across the street from the restaurant you're going to.

    Using bitmap compression (RoaringBitmap) and cardinality estimatiom algorithms (HyperLogLog/HyperLogLog++) to optimize the query system for various time and space constraints presented by IoT/low power devices.

Animal Colony Management System

  • An animal data management system for medical researchers built with React.js, Node.Js, and Google Firestore.

    This application will prove beneficial for researchers that need streamlined access to animal colony data, sharing capabilities, photo upload capabilities, and note taking capabilities.

DFS: A distributed filesystem

  • A distributed filesystem built with Java, Netty, and Google Protocol Buffers.

    The filesystem supports replication, dynamic addition of storage nodes, and compression.

    The system is also fault tolerant in that if a node goes down or data is corrupted the original data is still available.

    Bloom filters are used by the controller for probobalistic routing to data requests.

Fire-Engine: An in-memory search engine

  • A multi-threaded search engine that tracks user searches, allows users to crawl web pages, and search an inverted index built from crawled web pages.

itsh: A Command Line Shell

  • A command line shell that implements many features of other common command line shells.

    This includes but is not limited to history, bang execution, piping, redirection, background jobs, and arrow key support.
